!3 UseCase017 Diskussionskreis anlegen
!4 Aktor
 * Admin

!4 Stakeholder und deren Interessen:
 * Anwender hat einen neuen Diskussionskreis bestellt und will ihn umgehend nutzen können

!4 Vorbedingung
-- keine--

!4 Normaler Ablauf
 * Der Admin geht in das ControlPanel und wählt dort unter "Portal" das Feld "Communities" aus.
 * In Communities wählt er "hinzufügen".
 * Das System zeigt dem Admin eine Liste von Vorlagen an.
 * Der Admin wählt eine Vorlage aus.
   * Für einen normalen Diskussionskreis wählt er "Standard-DK"
   * Für eine BI wählt er "Standard-BI"
 * Der Admin 
   * vergibt den Namen, 
   * erstellt eine Beschreibung, 
   * schaltet die Propagation aus, 
   * setzt den Mitgliedschaftstyp auf "privat" und 
   * erstellt den DK.
 * Der Admin testet, ob der neue DK für unangemeldete Gäste auch dann unsichtbar ist, wenn man die URL kennt.
   * Falls nein, muss er die Berechtigung auf allen betroffenen Seiten entsprechend abändern.
